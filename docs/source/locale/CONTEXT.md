# locale/ — Sphinx i18n Translation Directory

## Intent
This directory contains the **internationalization (i18n) resources** for the EvoX documentation. It currently supports a single target language: **Simplified Chinese (zh_CN)**. The locale directory is wired into the Sphinx build via `locale_dirs = ["locale/"]` in `conf.py`, with a compacted gettext catalog (`gettext_compact = "docs"`). It is excluded from Sphinx's source scanning via `exclude_patterns = ["locale/**"]`.

The locale uses a **dual translation strategy**:

| Content Type | Mechanism | Location |
|---|---|---|
| **Tutorial pages** | Full-page manual translation | `zh_CN/tutorial/` (`.md` files) |
| **Everything else** (API docs, guides, examples, misc) | Gettext `.po` catalog, machine-translated | `zh_CN/LC_MESSAGES/docs.po` |

## Why Two Mechanisms?

The tutorials are **manually translated as full Markdown pages** rather than via the gettext pipeline. This ensures high-quality, natural-sounding Chinese prose for the onboarding experience. At build time, the `.readthedocs.yaml` pre-build job copies the translated tutorial `.md` files from `locale/zh_CN/tutorial/` into `docs/source/tutorial/`, overwriting the English originals. This works because Sphinx sees the Chinese pages as the original source — no gettext lookup is involved for tutorial content.

Everything else — API reference pages, installation guides, developer guides, examples, and miscellaneous pages — is translated via the standard Sphinx gettext mechanism. The single compacted `docs.po` catalog (~12,700 lines) maps every translatable string from the English source files to its Chinese equivalent.

## API Surface

```
locale/
└── zh_CN/                          # Simplified Chinese translations
    ├── LC_MESSAGES/
    │   └── docs.po                 # Gettext catalog: msgid (English) → msgstr (Chinese)
    └── tutorial/
        ├── index.md                # Tutorial landing page (Chinese)
        ├── tutorial_part1.md       # 1. 基础入门
        ├── tutorial_part2.md       # 2. ...
        ├── tutorial_part3.md
        ├── tutorial_part4.md
        ├── tutorial_part5.md
        ├── tutorial_part6.md
        └── tutorial_part7.md
```

### The `.po` Catalog (`docs.po`)

- **Format**: Standard GNU gettext `.po` file, generated by Babel 2.17.0
- **Size**: ~12,700 lines covering all translatable strings from the English documentation source
- **Source references**: Each entry includes `#: <path>:<line>` comments pointing back to the English source file (e.g., `#: ../../source/tutorial/tutorial_part1.md:1`)
- **Non-tutorial content**: Actively machine-translated; `msgstr` contains the Chinese translation
- **Tutorial content**: Deliberately left as English (msgstr = msgid) because the full-page manual translations take precedence
- **Fuzzy entries**: Marked with `#, fuzzy` flag; skipped by the machine translation pipeline until resolved

### Tutorial Translated Pages

- 8 files mirroring `source/tutorial/` exactly (same filenames, same structure)
- `index.md` contains a MyST toctree, just like the English version, but with a Chinese title (`# EvoX 教程`)
- Each `tutorial_partN.md` is a full manual translation — not a gettext extraction — with complete Chinese prose, translated headings, and preserved code blocks
- The English toctree in `conf.py` still works because the filename structure is identical

## Build-Time Integration

### `.readthedocs.yaml` Pre-Build Job

```
if [ "$READTHEDOCS_LANGUAGE" != "en" ]; then
    LANGUAGE_CODE="zh_CN"    # maps "zh-cn" (RTD format) → "zh_CN" (locale dir format)
    cp -r docs/source/locale/${LANGUAGE_CODE}/tutorial docs/source
fi
```

This copies the translated tutorials into `docs/source/tutorial/`, overwriting the English originals **before** Sphinx runs. Sphinx then processes the Chinese `.md` files as if they were the original source.

### Sphinx Configuration

- `locale_dirs = ["locale/"]` — Tells Sphinx where to find `.po` catalogs
- `gettext_compact = "docs"` — All translatable strings from all source files are compacted into a single `docs.po` per language (rather than one `.po` per source file)
- `exclude_patterns = ["locale/**"]` — Prevents the locale directory from being scanned as a documentation source
- Language switching via `READTHEDOCS_LANGUAGE` environment variable (set by Read the Docs); `conf.py` uses it for nav labels, theme context, and language-specific URLs

## Translation Maintenance Pipeline

The `.po` file is maintained through a pipeline of scripts in the parent `docs/` directory:

1. **`machine_translate.py`**: Fills in untranslated (`msgstr == ""`) and fuzzy entries by calling a GPT-4o API via TGI-compatible endpoint. Explicitly **skips**:
   - Code references matching `{py:obj}\`...\`` or `{py:mod}\`...\``
   - SVG content (`<svg ...>`)
   - Entries whose occurrences are **all** in `source/tutorial/` (tutorials are manually translated)
   
2. **`fix_output.py`**: Post-processes the `.po` file by removing entries where `msgid` contains "Duplicate implicit target name" — these are Sphinx build artifacts that should not appear in the translation catalog.

3. **`fix_notebook_translation.py`**: Wraps translated notebook strings as JSON cell structures in the `.po` file, reconciling how Sphinx's notebook builder expects translated notebook content.

## Constraints

- **Tutorial entries in `.po` MUST NOT be translated**: The `machine_translate.py` script enforces this by design. The full-page `.md` translations in `zh_CN/tutorial/` are the single source of truth for tutorial translations. If someone manually edits tutorial `msgstr` entries in the `.po`, those translations will be ignored (the copied `.md` files take precedence) and may cause confusion.
- **File structure must exactly mirror the English source**: The translated `.md` files in `zh_CN/tutorial/` must use the exact same filenames as the English originals in `source/tutorial/` for the `.readthedocs.yaml` copy step to work correctly.
- **Language code mapping**: Read the Docs uses lowercase-with-dashes (`zh-cn`), but the locale directory uses the standard POSIX locale format with underscores (`zh_CN`). The `.readthedocs.yaml` pre-build script handles this mapping.
- **gettext_compact = "docs"**: All translation strings are in a single `docs.po` file. When adding new documentation pages, the `.po` must be regenerated/updated (via `make gettext` + `sphinx-intl update`) to include new translatable strings.
- **No other languages currently supported**: Only `zh_CN` exists. Adding a new language would require: creating the locale subdirectory structure, translating the tutorial `.md` files, generating the `.po` catalog, and updating `.readthedocs.yaml`'s language code mapping.
