# locale/zh_CN — Simplified Chinese Localization

## Intent
This directory contains all **Simplified Chinese (zh_CN) translations** for the EvoX documentation. It is the sole non-English locale in the documentation system, enabling a fully bilingual (EN ↔ zh_CN) experience on Read the Docs.

## Structure

```
zh_CN/
├── LC_MESSAGES/
│   └── docs.po          ← Gettext translation catalog (~1,827 msgid entries, ~12,743 lines)
└── tutorial/
    ├── index.md          ← Tutorial landing page ("EvoX 教程")
    ├── tutorial_part1.md ← 1. 基础入门 (Basic Introduction)
    ├── tutorial_part2.md ← 2. 核心模块 (Core Modules)
    ├── tutorial_part3.md ← (Part 3)
    ├── tutorial_part4.md ← (Part 4)
    ├── tutorial_part5.md ← (Part 5, largest at 458 lines)
    ├── tutorial_part6.md ← (Part 6)
    └── tutorial_part7.md ← (Part 7)
```

## API Surface

### 1. Gettext Translation Catalog (`LC_MESSAGES/docs.po`)
- **Format**: Standard GNU gettext `.po` file, generated by Babel 2.17.0
- **Scope**: Translates autodoc-generated API labels (e.g., "Subpackages" → "子包", "Package Contents" → "软件包内容"), navigation strings, and UI elements embedded in the Sphinx build
- **Sphinx wiring**: `conf.py` sets `locale_dirs = ["locale/"]` and `gettext_compact = "docs"`, which maps to this file at build time
- **Source references**: Each entry includes `#: ../../source/apidocs/...` annotations pointing back to the original English source locations

### 2. Translated Tutorial Pages (`tutorial/`)
- **8 files** mirroring `docs/source/tutorial/` exactly (index.md + 7 numbered parts)
- **Manually translated**: These pages are Chinese prose translations of the English tutorials, *not* machine-generated
- **Content**: Covers the same getting-started topics: EvoX overview, core modules (Algorithm, Problem, Monitor, Workflow), GPU/distributed execution, functional API design, built-in algorithms/problems, and hands-on walkthroughs
- **Sphinx wiring**: `exclude_patterns = ["locale/**"]` in `conf.py` prevents these files from being processed as regular source documents; Sphinx's i18n machinery serves them when the language is set to zh_CN

## Relationship to the English Source

| English Source | zh_CN Translation |
|---|---|
| `docs/source/tutorial/*.md` | `docs/source/locale/zh_CN/tutorial/*.md` |
| Sphinx autodoc labels, nav strings | `LC_MESSAGES/docs.po` (msgid → msgstr) |

The `READTHEDOCS_LANGUAGE` environment variable (read in `conf.py`, defaulting to `"en"`) determines which language is active. When set to a non-English value:
- Nav labels and UI strings are pulled from the `.po` catalog
- Translated tutorial pages are served in place of their English counterparts

## Translation Maintenance Pipeline

Three scripts at `docs/` (one level up from this directory) manage the `.po` file:

| Script | Role |
|---|---|
| `machine_translate.py` | Calls a TGI-compatible LLM API (GPT-4o) to machine-translate untranslated `.po` entries from English → Chinese. Skips code references, tutorials (already manually translated), SVGs, and fuzzy entries. |
| `fix_output.py` | Post-processes the `.po` file to strip "Duplicate implicit target name" warnings that arise from Sphinx's gettext extraction |
| `fix_notebook_translation.py` | Wraps translated notebook strings as JSON cell structures in `.po` entries to satisfy the Sphinx notebook builder |

**Workflow**: `machine_translate.py` → `fix_output.py` → `fix_notebook_translation.py` → commit the updated `docs.po`.

## Constraints

- **Tutorial files are manually translated** — they must NOT be machine-translated or overwritten by the translation scripts
- **The `.po` file requires post-processing** after any machine translation run (run both fixup scripts)
- **`locale/**` is excluded from the Sphinx source tree** (`exclude_patterns` in `conf.py`) — files here are only consumed through Sphinx's i18n subsystem, not as direct source documents
- **This is the only locale directory** — there are no other language translations in the project
- **The `.po` file is tracked in git** and is maintained as part of the documentation source (not a build artifact)
- **Translation is also managed via Weblate** — the `Language-Team` header in `docs.po` references `https://hosted.weblate.org/projects/evox/evox/zh_Hans/`

## Routing Table

This is a leaf node with no further subdirectories requiring context extraction.
